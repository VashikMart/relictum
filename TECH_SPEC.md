# RELICTUM — техническая спецификация продакшн-сайта (v1.0)

Цель: перевести прототип V1 «The Gallery» в боевой сайт для рынка РФ с люксовым UX,
сильным SEO и удобным управлением контентом силами сотрудников. Документ — бриф для
передачи разработке (Opus/Claude Code или подрядчику).

## 1. Стек

| Слой | Выбор | Почему |
|---|---|---|
| Фронтенд | **Next.js 15 (App Router, RSC)** + TypeScript | SSR/ISR из коробки → SEO; образ у Stargift-класса сайтов |
| Стили | Tailwind CSS + токены из `01_design_system/tokens.css` | дизайн-система уже готова |
| CMS (контент) | **Payload CMS 3** (встраивается прямо в Next.js) — либо Directus, если нужен чистый self-host отдельно | админка как в `09_admin`: коллекции, роли, версии, черновики/публикация, локализация |
| БД | PostgreSQL (управляемый в Yandex Cloud / Selectel) | надёжно, РФ-хостинг |
| Файлы | S3-совместимое хранилище (Yandex Object Storage / Selectel) + CDN | фото 2K, видео 360° |
| Поиск каталога | Meilisearch (self-host) | мгновенный поиск «вид/место/эпоха», опечатки |
| Хостинг | VPS/K8s в РФ (Selectel / Timeweb Cloud / Yandex Cloud) за CDN | 152-ФЗ: персональные данные в РФ |
| Платежи/резерв | ЮKassa (депозит/резерв объекта), счёт для юрлиц | топ-лоты — оффлайн сделка, сайт делает «резерв» |
| Аналитика | Яндекс.Метрика + серверные события | рынок РФ |
| CRM | **amoCRM** (или Bitrix24) — двусторонняя интеграция | лучшие практики: лиды/сделки в CRM, товар/досье в CMS |

## 2. Модель контента (коллекции CMS)

- **Objects (экспонаты)** — поля как в `data/catalog.json` + галерея, видео 360°, статус
  (draft → review → published → reserved → sold → archive), «показ цены» (цена / по запросу /
  закрытый каталог), связанная оправа, связанные статьи. Версионирование + аудит-лог.
- **Provenance** — вложенная запись: экспертизы (файлы PDF), цепочка владения, QR-slug
  (`/dossier/R-0101` — публичное цифровое досье).
- **Worlds/Категории**, **Journal (статьи)**, **Expeditions (программы+даты)**,
  **Selections (подборки: повод/получатель/бюджет — приём luxpodarki)**,
  **Leads (заявки)** — форма сайта → CMS → вебхук в amoCRM,
  **Pages** (о доме, ритуал, доставка), **Team/Experts** (подписи под экспертизами).

## 3. Роли и процесс (как в 09_admin)

- Роли: Куратор (создаёт/редактирует), Эксперт (заполняет экспертизу), Фото-стандарт
  (чек фото), Главред (публикация), Консьерж (заявки), Админ.
- Публикация только через чек-лист: Provenance ✓, 2 экспертизы ✓, 5+ фото ✓, оправа ✓.
- Любое изменение — новая версия с автором; откат в 1 клик.
- AI-пайплайн текстов: черновик статьи/описания (Opus/Sonnet, промпт с брифом рубрики
  и данными объекта) → научный редактор → главред. Кнопка «Сгенерировать драфт» в CMS.

## 4. SEO (критично для РФ)

- SSR/ISR для всех страниц; карточка объекта — статически регенерируется при публикации.
- ЧПУ: `/catalog/cosmos/pallasit-seymchan-R-0101`; хлебные крошки + разметка.
- **Schema.org**: `Product` + `Offer` (или `priceSpecification` «по запросу»),
  `Article`, `BreadcrumbList`, `Organization`+`logo`, `FAQPage` на сервисных страницах.
- Мета из CMS (title/description на карточку — поле уже в админке), OG-картинки
  генерируются из фото объекта + лок-ап.
- sitemap.xml/robots автогенерация; RSS журнала; Яндекс.Вебмастер + turbo-совместимость.
- Core Web Vitals: next/image (AVIF/WebP), лениво всё ниже фолда, шрифты self-host
  (Cormorant Garamond + Inter локально, `font-display: swap`).
- Контент-SEO: журнал = органика («зуб мегалодона купить», «метеорит Сеймчан»),
  перелинковка статья ↔ 2–3 экспоната (поле в CMS).

## 5. Интеграции

- amoCRM: лид из форм/квиза → сделка; статусы «показ назначен/резерв/продан» синкают
  бейдж объекта на сайте.
- Telegram-бот дома: уведомление консьержу о заявке (SLA 24 ч), карточка лида.
- Резерв онлайн: депозит через ЮKassa переводит объект в `reserved` на 72 ч.
- Квиз «Персональный подбор» (5 вопросов, как консьерж luxpodarki) → лид с тегами.

## 6. Что уже готово к передаче

## 7. Этапы

1. **MVP (3–4 нед):** Next.js + Payload, каталог/карточка/заявки, SEO-база, деплой РФ.
2. **Фаза 2:** журнал+экспедиции, квиз-подбор, ЮKassa-резерв, amoCRM, поиск Meilisearch.
3. **Фаза 3:** закрытый каталог (личный кабинет по инвайту), цифровые досье QR,
   EN-версия для международных клиентов.
